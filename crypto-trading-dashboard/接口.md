# 加密货币交易 Dashboard API 接口文档

## 📋 概述

本文档描述了加密货币交易 Dashboard 前端所需的所有后端 API 接口。所有接口均采用 RESTful 设计，遵循以下规范：

- **基础URL**: `https://api.example.com/api`
- **请求方法**: GET / POST / PUT / DELETE
- **内容类型**: `application/json`
- **字符编码**: UTF-8

---

## 🔧 通用说明

### 请求头
```http
Content-Type: application/json
Accept: application/json
Authorization: Bearer <token>  // 需要认证时
```

### 通用响应格式
成功响应：
```json
{
  "data": {},  // 实际数据
  "status": 200
}
```

错误响应：
```json
{
  "message": "错误描述信息",
  "code": 400,
  "status": 400,
  "details": {}
}
```

### 错误码说明
- `200`: 成功
- `400`: 参数错误
- `401`: 未授权
- `404`: 资源不存在
- `500`: 服务器内部错误

## 📊 1. 市场数据接口

### 1.1 获取市场总览数据

**接口路径**: `/market/list` 和 `/market/overview`

**请求方式**: `GET`

**功能说明**: 返回所有主要加密货币的实时价格、涨跌幅、24小时交易量等市场概况信息

**请求参数**:
| 参数名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| symbol | string | 否 | - | 指定币种符号（如 'btc'），不传则返回所有 |
| vs_currency | string | 否 | usd | 法币类型，支持 usd, cny, eur 等 |

**请求示例**:
```http
GET /api/market/list?vs_currency=usd
GET /api/market/overview?symbol=btc&vs_currency=usd
```

**返回格式**:
```json
[
  {
    "id": "bitcoin",
    "symbol": "BTC",
    "name": "Bitcoin",
    "current_price": 45000.00,
    "price_change_24h": 1200.50,
    "price_change_percentage_24h": 2.74,
    "market_cap": 850000000000.00,
    "total_volume": 25000000000.00,
    "last_updated": "2025-11-03T10:00:00Z"
  }
]
```

**字段说明**:
- `id`: 币种唯一标识
- `symbol`: 币种符号（大写）
- `name`: 币种中文/英文名称
- `current_price`: 当前价格
- `price_change_24h`: 24小时价格变化
- `price_change_percentage_24h`: 24小时价格变化百分比
- `market_cap`: 市值
- `total_volume`: 24小时交易量
- `last_updated`: 最后更新时间

---

### 1.2 获取 K 线数据

**接口路径**: `/market/kline`

**请求方式**: `GET`

**功能说明**: 返回指定币种和时间周期的 OHLCV（开高低收成交量）数据，用于绘制K线图

**请求参数**:
| 参数名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| symbol | string | 是 | - | 币种符号（小写，如 'btc'） |
| interval | string | 否 | 1h | 时间间隔：1h(1小时), 4h(4小时), 1d(1天), 1w(1周) |
| start | number | 否 | - | 开始时间戳（毫秒） |
| end | number | 否 | - | 结束时间戳（毫秒） |
| limit | number | 否 | 100 | 返回数据条数，最大1000 |

**请求示例**:
```http
GET /api/market/kline?symbol=btc&interval=1h&limit=100
GET /api/market/kline?symbol=eth&interval=4h&start=1640995200000&end=1641081600000
```

**返回格式**:
```json
[
  {
    "time": 1640995200000,
    "open": 45000.00,
    "high": 46000.00,
    "low": 44000.00,
    "close": 45500.00,
    "volume": 125.50
  }
]
```

**字段说明**:
- `time`: 时间戳（毫秒）
- `open`: 开盘价
- `high`: 最高价
- `low`: 最低价
- `close`: 收盘价
- `volume`: 成交量

---

### 1.3 获取支撑阻力位

**接口路径**: `/market/support-resistance`

**请求方式**: `GET`

**功能说明**: 分析价格图表，识别关键支撑位和阻力位，用于交易决策参考

**请求参数**:
| 参数名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| symbol | string | 是 | - | 币种符号（小写，如 'btc'） |
| interval | string | 否 | 4h | 时间间隔：4h, 1d, 1w |

**请求示例**:
```http
GET /api/market/support-resistance?symbol=btc&interval=4h
```

**返回格式**:
```json
{
  "symbol": "BTC",
  "interval": "4h",
  "support_levels": [
    {
      "price": 44000.00,
      "strength": 0.85,
      "touches": 3
    },
    {
      "price": 43000.00,
      "strength": 0.72,
      "touches": 2
    }
  ],
  "resistance_levels": [
    {
      "price": 46000.00,
      "strength": 0.78,
      "touches": 2
    },
    {
      "price": 47000.00,
      "strength": 0.65,
      "touches": 1
    }
  ],
  "last_updated": "2025-11-03T10:00:00Z"
}
```

**字段说明**:
- `support_levels`: 支撑位数组
  - `price`: 支撑位价格
  - `strength`: 强度系数（0-1），数值越高表示该位置越重要
  - `touches`: 价格触及该位置的次数
- `resistance_levels`: 阻力位数组
  - `price`: 阻力位价格
  - `strength`: 强度系数（0-1）
  - `touches`: 价格触及该位置的次数

### 1.4 获取技术指标

**接口路径**: `/market/indicators`

**请求方式**: `GET`

**功能说明**: 计算并返回各种技术分析指标，帮助判断价格趋势和超买超卖状态

**请求参数**:
| 参数名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| symbol | string | 是 | - | 币种符号（小写，如 'btc'） |
| interval | string | 否 | 1h | 时间间隔 |
| indicators | string | 否 | rsi,macd,bollinger,ma | 指标类型，用逗号分隔 |

**请求示例**:
```http
GET /api/market/indicators?symbol=btc&interval=1h&indicators=rsi,macd
```

**返回格式**:
```json
{
  "symbol": "BTC",
  "interval": "1h",
  "rsi": {
    "value": 65.50,
    "signal": "neutral",
    "history": [45.20, 50.10, 55.30, 60.20, 65.50]
  },
  "macd": {
    "macd": 125.50,
    "signal": 118.20,
    "histogram": 7.30,
    "signal_type": "bullish"
  },
  "bollinger": {
    "upper": 46500.00,
    "middle": 45000.00,
    "lower": 43500.00,
    "position": 0.65
  },
  "ma": {
    "ma20": 44800.00,
    "ma50": 44200.00,
    "ma200": 42000.00,
    "trend": "uptrend"
  }
}
```

**字段说明**:

1. **RSI 指标 (相对强弱指数)**
   - `value`: RSI 当前值（0-100）
   - `signal`: 信号类型
     - `overbought`: 超买（>70）
     - `oversold`: 超卖（<30）
     - `neutral`: 中性（30-70）
   - `history`: 历史 RSI 值数组

2. **MACD 指标 (指数平滑异同移动平均线)**
   - `macd`: MACD 线值
   - `signal`: 信号线值
   - `histogram`: 柱状图值
   - `signal_type`: 信号类型
     - `bullish`: 看涨
     - `bearish`: 看跌

3. **布林带指标 (Bollinger Bands)**
   - `upper`: 布林带上轨
   - `middle`: 布林带中轨（20日移动平均）
   - `lower`: 布林带下轨
   - `position`: 当前价格在布林带中的位置（0-1）

4. **移动平均线指标 (Moving Average)**
   - `ma20`: 20日移动平均线
   - `ma50`: 50日移动平均线
   - `ma200`: 200日移动平均线
   - `trend`: 趋势方向
     - `uptrend`: 上升趋势
     - `downtrend`: 下降趋势
     - `sideways`: 横盘整理

### 1.5 获取趋势数据

**接口路径**: `/market/trend`

**请求方式**: `GET`

**功能说明**: 返回价格趋势分析数据，支持多币种对比和自定义时间范围

**请求参数**:
| 参数名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| symbol | string | 否 | - | 币种符号（小写），不传则返回多币种数据 |
| start_date | string | 否 | - | 开始日期（YYYY-MM-DD 格式） |
| end_date | string | 否 | - | 结束日期（YYYY-MM-DD 格式） |
| interval | string | 否 | 1d | 数据间隔：1h, 1d, 1w |

**请求示例**:
```http
GET /api/market/trend?symbol=btc&start_date=2025-01-01&end_date=2025-11-03&interval=1d
GET /api/market/trend?interval=1w
```

**返回格式**:
```json
{
  "data_range": {
    "start": "2025-01-01",
    "end": "2025-11-03"
  },
  "time_interval": "1d",
  "coins": [
    {
      "symbol": "BTC",
      "prices": [
        {
          "timestamp": 1640995200000,
          "price": 45000.00,
          "date": "2025-01-01"
        }
      ],
      "trend": "upward",
      "volatility": 0.045,
      "avg_price": 45500.00
    }
  ],
  "market_summary": {
    "best_performer": "ETH",
    "worst_performer": "DOGE",
    "total_change": 12.50
  }
}
```

**字段说明**:
- `data_range`: 数据时间范围
- `time_interval`: 数据间隔
- `coins`: 币种数据数组
  - `prices`: 价格数据
    - `timestamp`: 时间戳（毫秒）
    - `price`: 价格
    - `date`: 日期（用户本地时间格式）
  - `trend`: 整体趋势
    - `upward`: 上升趋势
    - `downward`: 下降趋势
    - `sideways`: 横盘整理
  - `volatility`: 波动率（标准差）
  - `avg_price`: 平均价格
- `market_summary`: 市场摘要
  - `best_performer`: 表现最佳币种
  - `worst_performer`: 表现最差币种
  - `total_change`: 市场总体变化百分比

## 📝 使用说明

### 时间处理
- 所有时间戳均使用秒级时间戳（如：1640995200）
- 日期使用 YYYY-MM-DD 格式（如：2025-01-01）
- 时间戳会自动转换为用户本地时间显示

### 币种符号
- 后端 API 接收小写符号（如："btc", "eth"）
- 前端会处理大小写转换，返回时统一大写

### 数值精度
- 所有价格数据保持 2-4 位小数
- 百分比数据保持 2 位小数
- 比率数据保持 3-4 位小数

### 数据限制
- K线数据最大返回 1000 条
- 趋势数据最大支持 1 年时间范围
- 技术指标历史数据默认返回 50 个点

## 🛠️ 开发建议

### 缓存策略
- 市场总览数据：缓存 30 秒
- K线数据：缓存 10 秒
- 技术指标：缓存 1 分钟
- 趋势数据：缓存 5 分钟

### 错误处理
- 建议实现重试机制（最多 3 次）
- 网络错误应显示友好提示
- 数据为空时应显示 "暂无数据" 提示

### 性能优化
- 支持分页查询（大型数据集）
